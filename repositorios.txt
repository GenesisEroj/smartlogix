REPOSITORIO PRINCIPAL
---------------------
Nombre: smartlogix
URL: https://github.com/GenesisEroj/smartlogix
Descripción: Repositorio principal del proyecto. Contiene la documentación
general, análisis de patrones arquitectónicos, estrategia de branching y
enlaces a los repositorios de cada componente del sistema.

COMPONENTES
-----------

1. FRONTEND
Nombre: smartlogix-frontend
URL: https://github.com/GenesisEroj/smartlogix-frontend
Descripción: Componente frontend desarrollado como módulo NPM reutilizable
con React 18 + Vite. Incluye la interfaz de usuario para la gestión de
pedidos e inventario, implementando los patrones Factory Method (api.js),
Observer (useNotifications.js) y Repository (useInventario.js, usePedidos.js).

2. BACKEND FOR FRONTEND (BFF)
Nombre: smartlogix-bff
URL: https://github.com/GenesisEroj/smartlogix-bff
Puerto: 8084
Descripción: Intermediario entre el frontend y los microservicios desarrollado
con Spring Boot 3.3.5. Gestiona y consolida las respuestas de MS Pedidos y
MS Inventario, implementando los patrones Factory Method (WebClientConfig.java)
y Strategy (InventarioService.java, PedidosService.java).

3. MICROSERVICIO PEDIDOS
Nombre: smartlogix-ms-pedidos
URL: https://github.com/GenesisEroj/smartlogix-ms-pedidos
Puerto: 8081
Descripción: Microservicio de Pedidos desarrollado con Spring Boot 3.3.5 y
Spring Data JPA. Gestiona la creación y seguimiento de pedidos implementando
los patrones Factory Method, CQRS y Repository. Conectado a MySQL mediante JPA.

4. MICROSERVICIO INVENTARIO
Nombre: smartlogix-ms-inventario
URL: https://github.com/GenesisEroj/smartlogix-ms-inventario
Puerto: 8082
Descripción: Microservicio de Inventario desarrollado con Spring Boot 3.3.5 y
Spring Data JPA. Gestiona el stock de productos implementando los patrones
Repository, CQRS y DTO. Conectado a MySQL mediante JPA.

5. ARQUETIPO MAVEN
Nombre: smartlogix-archetype
URL: https://github.com/GenesisEroj/smartlogix-archetype
Descripción: Arquetipo Maven personalizado que genera la estructura base de
cualquier microservicio del proyecto con Spring Boot + JPA + MySQL
preconfigurado. Usado como base para generar ms-inventario y ms-pedidos.
